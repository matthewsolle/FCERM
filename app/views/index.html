Version 1 design history

Main research objectives

Understand

  • Each participants context of use e.g. commercial use of research, academic use of research, EA/NRW use of research
  • How the participants context of use affects the way the access and use research

Establish, in the light of the participants context of use, are the following items usable (effective and efficient) to use:

  • Keyword searching
  • Filtering search results
  • Date and status filtering
  • Search results
  • Project meta data
  • Access to project documentation
  • Project summary

Additionally, determine the value to participants of:

  • The most appropriate style of date to associate with an inflight project delivery
  • Information about the FCERM project pipeline (projects / topics that haven’t been allocated funding)

Research findings

Finder

  • Keyword search was effective and usable
  • Filtering by the combination of grouping/topic was ineffective and inefficient. Participants indicated that the grouping/topic filters looked like menu based navigation “where will this take me?”.
  • Participants were not confident what the proposed filters were or how to use them
  • The combined grouping/topic filters didn’t support participants desired search techniques e.g. “I want to search more broadly so I can be sure I haven’t missed anything” and “I want to find the one specific project that I remember reading about”
  • Topics duplicated across groupings led to confusion

Project page

  • Project summaries were successfully used to determine the value of a project
  • Projects need to display: 1. the project code 2. the project completion date to determine chronological relevance 3. a separate publication date and information about changes to the page/project 4. document type and file size, to provide additional context for the document “ is it a big file? Can I read it now? Or should I set aside time later.”

Prototype limitations

  • When filters are selected, the search control needs to be selected to display the results (see a live version of a Specialist Publisher finder to see how it will actually work https://www.gov.uk/raib-reports)
  • More than one filter can be selected (but to work they have to be selected at the same time)
  • Search terms work on report title but not on filter
  • To clear filters and return to default view (no filters selected), select search control again
  • Prototype currently only displaying one tag per filter. With the live Finder there will be no limitation how many tags for each filter can be displayed
  • The prototype currently only contains a small proportion of actual FCERM reports. 25 was thought to be representative and allow testing of the functionality
